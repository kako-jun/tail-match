# Tail Match — 動作環境・対応状況

## ブラウザ対応

モダンブラウザであれば動作します。スマートフォン・タブレットにも対応しています。

| 環境 | 状況 |
|------|------|
| Chrome（最新） | 対応 |
| Firefox（最新） | 対応 |
| Safari（最新） | 対応 |
| Edge（最新） | 対応 |
| スマートフォン（iOS Safari） | 対応 |
| スマートフォン（Android Chrome） | 対応 |

JavaScript が必要です。ブラウザの JavaScript を無効にしている場合は動作しません。

## Docker 環境（開発・本番共通）

開発環境・本番環境ともに Docker Compose で管理しています。

**構成コンテナ：**
- フロントエンド（Next.js）
- データベース（PostgreSQL）
- リバースプロキシ（Nginx）
- スクレイパー（Python）

開発環境と本番環境は別の Compose ファイルで管理し、設定を分離しています。

## スクレイパー対応状況

自治体サイトの構造はさまざまです。対応している取得方式を以下に示します。

### 静的 HTML サイト

HTMLをダウンロードしてそのまま解析します。最もシンプルな構造です。テーブル形式・カード形式・リスト形式のいずれにも対応しています。

### JavaScript 必須サイト（動的コンテンツ）

ページの内容が JavaScript で生成される自治体サイトに対応しています。ブラウザを自動操作することで、通常の閲覧と同じ状態のHTMLを取得してから解析します。待機時間を設けて、コンテンツの読み込みが完了してから取得します。

### 特殊な構造への対応

各自治体の独自形式にも個別対応しています：

- iframe 内にデータが入っているサイト（iframe URL を直接取得）
- 全角数字を使うサイト（半角に正規化）
- rowspan がある表（複数行にまたがるセルの解釈）
- 犬猫混在のページ（caption や URL パラメータで猫だけを識別）
- 複数タブで同一動物が重複する場合（ID 基準で重複除去、猫を優先保持）

### 未自動化の施設

一部の施設は、情報が画像・PDF・手書きスキャンの形式で公開されており、自動収集が困難な状態です。対応方法を継続検討中です。
